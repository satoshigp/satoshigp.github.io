  • スクリプト
    • transition(トランジション)とは？
    transitionとは？
    要素名A,クラス名{transition:-property-delay-duration-timing-function}
    transitionとは変化ということです。
    例）画像にマウスを乗せたら画像が大きくる等

    引数について

    property:transitionを適応するプロパティの指定
    all transitionが適応可能なプロパティすべてに効果を適応。
    none どのプロパティにも適応させない
    delay:変化が開始されるまでの時間
    任意の時間+単位単位はms(ミリ秒)、ｓ(秒)
    duration:開始から終了までの時間
    任意の時間+単位 単位はms(ミリ秒)、ｓ(秒)
    timing-function：変化の進行度
    ease 変化の開始と終了付近の動きを滑らかに。
    linear 一定で変化します
    ease-in 変化開始付近の動きを緩やかにします
    ease-out 変化終了付近の動きを緩やかにします
    ease-in-out 変化の開始と終了付近の動きを緩やかにします
    cubic-bezier()任意で変化量を決めることができます。
    回転(Quaternion.RotateTowards)
    Quaternion.RotateTowardsとは？
    Quaternion.RotateTowards(Quaternion A,Quaternion B,float X)
    こちらも開始Aから終点Bまでの間をX秒で移動するということ。
    こちらもこの方法で行うと一定数の回転が可能になる！ 　
  • オイラー角とクォータニオンとの違いが紛らわしい
  • クォータニオンで編集せずオイラー角で！
  • Unityのインスペクター上では360°以上を越せれるが実際は0～360°まで
  • オイラー角で編集を行うと0.000001単位(0が多かったような少なかったような...)で誤差が出る
  • RotateTowardsで回転する
    • 問題・課題

    ー問題ー

    回転により前進の向きの動きを変える

    ー解決方法ー
  • 前進後進がZ座標、左右がX座標。
  • 自身が回転してもZ座標で進んでしまう。
  • 私はswitch文で切り替えました！
  • 方法はスクリプト文を見ていただければ！

    • ー課題ー
    コルーチンを使ったやり方があるらしい！
